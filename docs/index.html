Introduction

    Introduction

    This documentation describes the Cypress automated test environment for OASys, covering the following information:

    • Writing and running tests - how to structure a test file, where to put it, how to run it and how to get results.
    • Function library
    • Example offender definitions
    • OASys pages
    • Standard test users
    • RestAPI testing

    The test environment contains the following key folders:

    • cypress - contains supporting functionality and some modified Cypress commands.
    • oasys - this main folder contains the various libraries described in this documentation:
      • type and class definitions
      • environments
      • lib
      • offenderLib
      • pages
      • populate
      • restApi
      • sns
      • users
    • tests - all test scripts and associated test data.

    Configuration

    The top-level folder contains a configuration file called localSettings.ts, this is used to define local environment configuration (such as selecting a target OASys environment). Comments in the config file define the settings available.

    Standard Cypress

    The OASys framework described in this documentation can be used for most things, but the standard Cypress functionality is also available for anything not covered by the framework. Some of the basic Cypress commands include:

    • cy.get(selector) to locate an element
    • cy.find(selector) to find a descendent element
    • cy.type(text) to enter text

    Commands can be chained together, e.g. cy.get('input[type='button'][value='Create Assessment']).click()
